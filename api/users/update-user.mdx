---
title: "Update User"
api: "PATCH /users/:id"
description: "Update an existing user's information"
---

This endpoint allows administrators to update user account information. All fields are optional.

## Authentication

<Warning>This endpoint requires Admin role authentication.</Warning>

Include the JWT token in the Authorization header:

```
Authorization: Bearer <your_token>
```

## Path Parameters

<ParamField path="id" type="string" required>
  The unique user identifier (UUID)
</ParamField>

## Request Body

All fields are optional. Only include the fields you want to update.

<ParamField body="name" type="string">
  User's first name (minimum 2 characters)
</ParamField>

<ParamField body="lastName" type="string">
  User's last name (minimum 2 characters)
</ParamField>

<ParamField body="email" type="string">
  User's primary email address (must be valid email format and unique)
</ParamField>

<ParamField body="password" type="string">
  User's new password (minimum 8 characters)
</ParamField>

<ParamField body="phone" type="string">
  User's phone number in international format (minimum 10 characters)
</ParamField>

<ParamField body="country" type="string">
  User's country code (e.g., "CO", "US", "MX")
</ParamField>

<ParamField body="language" type="string">
  User's preferred language (e.g., "es-Es", "en-US")
</ParamField>

<ParamField body="authProvider" type="enum">
  Authentication provider: `LOCAL`, `GOOGLE`, `FACEBOOK`, or `OTHER`
</ParamField>

<ParamField body="role" type="enum">
  User's role: `admin` or `cliente`
</ParamField>

## Response

Returns the updated user object with all its properties.

<ResponseField name="user_id" type="string">
  Unique user identifier (UUID)
</ResponseField>

<ResponseField name="name" type="string">
  User's first name
</ResponseField>

<ResponseField name="lastName" type="string">
  User's last name
</ResponseField>

<ResponseField name="email" type="string">
  User's primary email address
</ResponseField>

<ResponseField name="phone" type="string">
  User's phone number
</ResponseField>

<ResponseField name="country" type="string">
  User's country code
</ResponseField>

<ResponseField name="language" type="string">
  User's preferred language
</ResponseField>

<ResponseField name="role" type="enum">
  User's role
</ResponseField>

<ResponseField name="status" type="enum">
  User's account status
</ResponseField>

<ResponseField name="updateAt" type="string">
  Last update timestamp (updated automatically)
</ResponseField>

## Example Request

<CodeGroup>
```bash cURL
curl -X PATCH https://api.rentapelis.com/users/123e4567-e89b-12d3-a456-426614174000 \
  -H "Authorization: Bearer <your_token>" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Juan Carlos",
    "phone": "+573009876543",
    "language": "en-US"
  }'
```

```javascript JavaScript
const userId = '123e4567-e89b-12d3-a456-426614174000';
const response = await fetch(`https://api.rentapelis.com/users/${userId}`, {
  method: 'PATCH',
  headers: {
    'Authorization': 'Bearer <your_token>',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    name: 'Juan Carlos',
    phone: '+573009876543',
    language: 'en-US'
  })
});
const user = await response.json();
console.log(user);
```

```python Python
import requests

user_id = '123e4567-e89b-12d3-a456-426614174000'
headers = {
    'Authorization': 'Bearer <your_token>',
    'Content-Type': 'application/json'
}

data = {
    'name': 'Juan Carlos',
    'phone': '+573009876543',
    'language': 'en-US'
}

response = requests.patch(
    f'https://api.rentapelis.com/users/{user_id}',
    headers=headers,
    json=data
)
user = response.json()
print(user)
```
</CodeGroup>

## Error Responses

<ResponseField name="statusCode" type="number">
  HTTP status code
</ResponseField>

<ResponseField name="message" type="string | array">
  Error message description or array of validation errors
</ResponseField>

### 400 Bad Request

Validation errors will return an array of messages:

```json
{
  "statusCode": 400,
  "message": [
    "email must be an email",
    "password must be longer than or equal to 8 characters"
  ],
  "error": "Bad Request"
}
```

### 401 Unauthorized

```json
{
  "statusCode": 401,
  "message": "Unauthorized"
}
```

### 403 Forbidden

Returned when the authenticated user does not have admin role:

```json
{
  "statusCode": 403,
  "message": "Forbidden resource"
}
```

### 404 Not Found

```json
{
  "statusCode": 404,
  "message": "User not found"
}
```

### 409 Conflict

Returned when trying to update email to one that already exists:

```json
{
  "statusCode": 409,
  "message": "Email already in use by another user"
}
```

### 500 Internal Server Error

```json
{
  "statusCode": 500,
  "message": "Internal server error"
}
```
