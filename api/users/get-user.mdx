---
title: "Get User"
api: "GET /users/:id"
description: "Retrieve detailed information about a specific user"
---

This endpoint returns detailed information about a user by their ID. This is an admin-only endpoint.

## Authentication

<Warning>This endpoint requires Admin role authentication.</Warning>

Include the JWT token in the Authorization header:

```
Authorization: Bearer <your_token>
```

## Path Parameters

<ParamField path="id" type="string" required>
  The unique user identifier (UUID)
</ParamField>

## Response

<ResponseField name="user_id" type="string" required>
  Unique user identifier (UUID)
</ResponseField>

<ResponseField name="name" type="string" required>
  User's first name
</ResponseField>

<ResponseField name="lastName" type="string" required>
  User's last name
</ResponseField>

<ResponseField name="email" type="string" required>
  User's primary email address (unique)
</ResponseField>

<ResponseField name="phone" type="string">
  User's phone number
</ResponseField>

<ResponseField name="backupEmail" type="string">
  User's backup email address
</ResponseField>

<ResponseField name="country" type="string">
  User's country code (default: "CO")
</ResponseField>

<ResponseField name="language" type="string">
  User's preferred language (default: "es-Es")
</ResponseField>

<ResponseField name="role" type="enum" required>
  User's role: `admin` or `cliente`
</ResponseField>

<ResponseField name="avatar" type="string">
  URL to user's avatar image
</ResponseField>

<ResponseField name="status" type="enum" required>
  User's account status: `ACTIVE`, `INACTIVE`, `DRAFT`, or `BLOCKED`
</ResponseField>

<ResponseField name="authProvider" type="enum" required>
  Authentication provider: `LOCAL`, `GOOGLE`, `FACEBOOK`, or `OTHER`
</ResponseField>

<ResponseField name="createdAt" type="string" required>
  Account creation timestamp (ISO 8601 format)
</ResponseField>

<ResponseField name="updateAt" type="string" required>
  Last update timestamp (ISO 8601 format)
</ResponseField>

<ResponseField name="phoneConfirm" type="boolean" required>
  Whether phone number is confirmed
</ResponseField>

<ResponseField name="emailConfirm" type="boolean" required>
  Whether primary email is confirmed
</ResponseField>

<ResponseField name="backupEmailConfirm" type="boolean" required>
  Whether backup email is confirmed
</ResponseField>

<ResponseField name="twoFactorEnable" type="boolean" required>
  Whether two-factor authentication is enabled
</ResponseField>

## Example Request

<CodeGroup>
```bash cURL
curl https://api.rentapelis.com/users/123e4567-e89b-12d3-a456-426614174000 \
  -H "Authorization: Bearer <your_token>"
```

```javascript JavaScript
const userId = '123e4567-e89b-12d3-a456-426614174000';
const response = await fetch(`https://api.rentapelis.com/users/${userId}`, {
  headers: {
    'Authorization': 'Bearer <your_token>'
  }
});
const user = await response.json();
console.log(user);
```

```python Python
import requests

user_id = '123e4567-e89b-12d3-a456-426614174000'
headers = {
    'Authorization': 'Bearer <your_token>'
}

response = requests.get(
    f'https://api.rentapelis.com/users/{user_id}',
    headers=headers
)
user = response.json()
print(user)
```
</CodeGroup>

## Error Responses

<ResponseField name="statusCode" type="number">
  HTTP status code
</ResponseField>

<ResponseField name="message" type="string">
  Error message description
</ResponseField>

### 400 Bad Request

```json
{
  "statusCode": 400,
  "message": "Invalid user ID format"
}
```

### 401 Unauthorized

```json
{
  "statusCode": 401,
  "message": "Unauthorized"
}
```

### 403 Forbidden

Returned when the authenticated user does not have admin role:

```json
{
  "statusCode": 403,
  "message": "Forbidden resource"
}
```

### 404 Not Found

```json
{
  "statusCode": 404,
  "message": "User not found"
}
```

### 500 Internal Server Error

```json
{
  "statusCode": 500,
  "message": "Internal server error"
}
```
