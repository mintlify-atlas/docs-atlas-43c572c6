---
title: "Delete User"
api: "DELETE /users/:id"
description: "Delete a user account from the system"
---

This endpoint allows administrators to permanently delete a user account. This action will cascade delete all related data including sessions, favorites, reviews, and orders.

## Authentication

<Warning>This endpoint requires Admin role authentication.</Warning>

Include the JWT token in the Authorization header:

```
Authorization: Bearer <your_token>
```

## Path Parameters

<ParamField path="id" type="string" required>
  The unique user identifier (UUID) of the user to delete
</ParamField>

## Response

Returns a success message when the user is deleted successfully.

<ResponseField name="message" type="string">
  Confirmation message
</ResponseField>

<ResponseField name="user_id" type="string">
  The ID of the deleted user
</ResponseField>

## Example Request

<CodeGroup>
```bash cURL
curl -X DELETE https://api.rentapelis.com/users/123e4567-e89b-12d3-a456-426614174000 \
  -H "Authorization: Bearer <your_token>"
```

```javascript JavaScript
const userId = '123e4567-e89b-12d3-a456-426614174000';
const response = await fetch(`https://api.rentapelis.com/users/${userId}`, {
  method: 'DELETE',
  headers: {
    'Authorization': 'Bearer <your_token>'
  }
});
const result = await response.json();
console.log(result);
```

```python Python
import requests

user_id = '123e4567-e89b-12d3-a456-426614174000'
headers = {
    'Authorization': 'Bearer <your_token>'
}

response = requests.delete(
    f'https://api.rentapelis.com/users/{user_id}',
    headers=headers
)
result = response.json()
print(result)
```
</CodeGroup>

## Example Response

```json
{
  "message": "User deleted successfully",
  "user_id": "123e4567-e89b-12d3-a456-426614174000"
}
```

## Cascade Deletion

<Warning>
  Deleting a user will permanently remove all associated data:
  - All active sessions
  - Favorite movies
  - Movie reviews
  - Orders and order items
  
  This action cannot be undone.
</Warning>

## Best Practices

<Tip>
  Instead of deleting users, consider:
  - Setting the user's `status` to `BLOCKED` or `INACTIVE` using the [Update User](/api/users/update-user) endpoint
  - This preserves data integrity and allows for potential account recovery
  - You can still prevent login by checking the status field during authentication
</Tip>
