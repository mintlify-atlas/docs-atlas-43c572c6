---
title: "List All Orders"
api: "GET /orders"
description: "Retrieve all orders in the system (Admin only)"
---

## Authentication

This endpoint requires:
- **Bearer Token**: Valid JWT access token
- **Role**: `Admin` role required

<Warning>
This endpoint is restricted to administrators only. Regular clients should use the [My Orders](/api/orders/my-orders) endpoint to view their own orders.
</Warning>

## Query Parameters

<ParamField query="status" type="string" optional>
  Filter orders by status. Must be one of:
  - `PENDING` - Orders pending payment
  - `PAID` - Paid orders
  - `CANCELED` - Canceled orders
  - `REFUNDED` - Refunded orders
  
  If omitted, returns orders with all statuses.
</ParamField>

## Response

Returns an array of order objects.

<ResponseField name="orderId" type="string">
  Unique identifier for the order (UUID format)
</ResponseField>

<ResponseField name="userId" type="string">
  ID of the user who created the order
</ResponseField>

<ResponseField name="totalAmount" type="number">
  Total cost of the order
</ResponseField>

<ResponseField name="status" type="string">
  Current order status: `PENDING`, `PAID`, `CANCELED`, or `REFUNDED`
</ResponseField>

<ResponseField name="createdAt" type="string">
  ISO 8601 timestamp when the order was created
</ResponseField>

<ResponseField name="updatedAt" type="string">
  ISO 8601 timestamp when the order was last updated
</ResponseField>

<ResponseField name="items" type="array">
  Array of order items
  
  <ResponseField name="id" type="string">
    Unique identifier for the order item
  </ResponseField>
  
  <ResponseField name="orderId" type="string">
    Reference to the parent order
  </ResponseField>
  
  <ResponseField name="imdbId" type="string">
    IMDB ID of the movie
  </ResponseField>
  
  <ResponseField name="price" type="number">
    Price of the item at the time of order creation
  </ResponseField>
  
  <ResponseField name="movie" type="object">
    Movie details
    
    <ResponseField name="title" type="string">
      Movie title
    </ResponseField>
    
    <ResponseField name="poster" type="string">
      URL to the movie poster image
    </ResponseField>
  </ResponseField>
</ResponseField>

## Example Request

```bash
# Get all orders
GET /orders

# Get only pending orders
GET /orders?status=PENDING

# Get only paid orders
GET /orders?status=PAID
```

## Behavior

- Orders are returned sorted by creation date (most recent first)
- Each order includes full movie details (title and poster) for all items
- Status filter is optional and case-sensitive
- Empty array is returned if no orders match the criteria

## Error Responses

<ResponseField name="401 Unauthorized" type="error">
  Returned when the access token is missing or invalid
  
  ```json
  {
    "statusCode": 401,
    "message": "Unauthorized"
  }
  ```
</ResponseField>

<ResponseField name="403 Forbidden" type="error">
  Returned when the user does not have the `Admin` role
  
  ```json
  {
    "statusCode": 403,
    "message": "Forbidden resource",
    "error": "Forbidden"
  }
  ```
</ResponseField>
