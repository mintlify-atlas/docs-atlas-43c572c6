## Authentication

No authentication required. This is a public endpoint.

## Request Body

<ParamField body="email" type="string" required>
  User's email address. Must be a valid email format. Will be automatically converted to lowercase and trimmed.
</ParamField>

<ParamField body="password" type="string" required>
  User's password. Minimum length: 6 characters.
</ParamField>

## Response

<ResponseField name="access_token" type="string">
  JWT access token for authenticating subsequent requests.
</ResponseField>

## Example Request

<CodeGroup>

```bash cURL
curl -X POST https://api.rentapelis.com/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "email": "juan.perez@example.com",
    "password": "SecurePass123!"
  }'
```

```javascript JavaScript
const response = await fetch('https://api.rentapelis.com/auth/login', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    email: 'juan.perez@example.com',
    password: 'SecurePass123!'
  })
});

const data = await response.json();
console.log(data);

// Store the token for future requests
localStorage.setItem('accessToken', data.access_token);
```

```python Python
import requests

response = requests.post(
    'https://api.rentapelis.com/auth/login',
    json={
        'email': 'juan.perez@example.com',
        'password': 'SecurePass123!'
    }
)

data = response.json()
print(data)
```

</CodeGroup>

## Error Codes

| Status Code | Description |
|-------------|-------------|
| 200 | Login successful, access token returned |
| 400 | Bad Request - Invalid input data (validation errors) |
| 401 | Unauthorized - Invalid credentials (user not found or incorrect password) |
| 500 | Internal Server Error |

## Using the Access Token

Once you receive the access token, include it in the `Authorization` header for all protected endpoints:

```bash
curl -X GET https://api.rentapelis.com/auth/profile \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN"
```

## Token Details

The JWT token contains the following payload:

- `sub` - User ID
- `email` - User's email address
- `role` - User's role (admin or cliente)
- `iat` - Issued at timestamp
- `exp` - Expiration timestamp

## Notes

- Email addresses are automatically converted to lowercase and trimmed of whitespace
- Passwords are trimmed before comparison for security
- The access token should be stored securely (e.g., in localStorage or secure cookies)
- Tokens have an expiration time and need to be refreshed periodically
- Failed login attempts should be monitored to prevent brute force attacks
