---
title: "Register User"
api: "POST /auth/register"
description: "Create a new user account and receive an authentication token"
---

## Authentication

No authentication required. This is a public endpoint.

## Request Body

<ParamField body="name" type="string" required>
  User's first name. Minimum length: 2 characters.
</ParamField>

<ParamField body="lastName" type="string" required>
  User's last name. Minimum length: 2 characters.
</ParamField>

<ParamField body="email" type="string" required>
  User's email address. Must be a valid email format. Will be automatically converted to lowercase and trimmed.
</ParamField>

<ParamField body="password" type="string" required>
  User's password. Minimum length: 8 characters.
</ParamField>

<ParamField body="phone" type="string" required>
  User's phone number. Must be a valid phone number format. Minimum length: 10 characters.
</ParamField>

<ParamField body="country" type="string" required>
  User's country.
</ParamField>

<ParamField body="language" type="string" required>
  User's preferred language.
</ParamField>

<ParamField body="authProvider" type="string" required>
  Authentication provider used for registration. Valid values:
  - `LOCAL` - Standard email/password registration
  - `GOOGLE` - Google OAuth
  - `FACEBOOK` - Facebook OAuth
  - `OTHER` - Other authentication providers
</ParamField>

<ParamField body="role" type="string" optional>
  User role. Defaults to `cliente` if not specified. Valid values:
  - `admin` - Administrator role
  - `cliente` - Client/customer role (default)
</ParamField>

## Response

<ResponseField name="accessToken" type="string">
  JWT access token for authenticating subsequent requests.
</ResponseField>

<ResponseField name="user" type="object">
  User information object.
  
  <ResponseField name="user.id" type="string">
    Unique user identifier.
  </ResponseField>
  
  <ResponseField name="user.email" type="string">
    User's email address.
  </ResponseField>
</ResponseField>

## Error Codes

| Status Code | Description |
|-------------|-------------|
| 201 | User successfully created and registered |
| 400 | Bad Request - Invalid input data (validation errors) |
| 409 | Conflict - Email already registered |
| 500 | Internal Server Error - Failed to create user |

## Notes

- Email addresses are automatically converted to lowercase and trimmed of whitespace
- The password is hashed before storage using a secure hashing algorithm
- The returned `accessToken` should be included in the `Authorization` header for protected endpoints
- Token format: `Bearer {accessToken}`
- Default user role is `cliente` if not specified
- Phone numbers must be in valid international format
