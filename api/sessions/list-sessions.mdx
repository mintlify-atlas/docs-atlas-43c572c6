---
title: "List User Sessions"
api: "GET /sessions/:id"
description: "Retrieve all sessions for a specific user"
---

This endpoint returns all sessions associated with a user ID. Use this to view all active and inactive sessions for a user.

## Authentication

<Info>Authentication requirements depend on your application's security policies.</Info>

## Path Parameters

<ParamField path="id" type="string" required>
  The unique user identifier (UUID) to retrieve sessions for
</ParamField>

## Response

<ResponseField name="sessions" type="array">
  Array of session objects for the user
  
  <Expandable title="Session Object">
    <ResponseField name="id" type="string" required>
      Unique session identifier (UUID)
    </ResponseField>
    
    <ResponseField name="userId" type="string" required>
      The user ID associated with this session
    </ResponseField>
    
    <ResponseField name="refreshToken" type="string" required>
      The refresh token for this session
    </ResponseField>
    
    <ResponseField name="userAgent" type="string">
      The user agent string of the client
    </ResponseField>
    
    <ResponseField name="ipAddress" type="string">
      The IP address of the client
    </ResponseField>
    
    <ResponseField name="location" type="string">
      Geographic location of the client
    </ResponseField>
    
    <ResponseField name="isActive" type="boolean" required>
      Whether the session is currently active
    </ResponseField>
    
    <ResponseField name="expiresAt" type="string">
      Session expiration timestamp (ISO 8601 format)
    </ResponseField>
    
    <ResponseField name="createdAt" type="string" required>
      Session creation timestamp (ISO 8601 format)
    </ResponseField>
    
    <ResponseField name="lastUsedAt" type="string" required>
      Last time the session was used (ISO 8601 format)
    </ResponseField>
  </Expandable>
</ResponseField>

## Example Request

<CodeGroup>
```bash cURL
curl https://api.rentapelis.com/sessions/123e4567-e89b-12d3-a456-426614174000
```

```javascript JavaScript
const userId = '123e4567-e89b-12d3-a456-426614174000';
const response = await fetch(`https://api.rentapelis.com/sessions/${userId}`);
const sessions = await response.json();
console.log(sessions);
```

```python Python
import requests

user_id = '123e4567-e89b-12d3-a456-426614174000'
response = requests.get(f'https://api.rentapelis.com/sessions/{user_id}')
sessions = response.json()
print(sessions)
```
</CodeGroup>

## Use Cases

This endpoint is useful for:

- **Security Dashboard**: Display all active sessions to the user
- **Device Management**: Show where the user is logged in
- **Session Monitoring**: Track suspicious login activity
- **Remote Logout**: List sessions before allowing user to revoke them

## Session Filtering

<Tip>
  To get only active sessions, filter the response array:
  
  ```javascript
  const activeSessions = sessions.filter(s => s.isActive);
  ```
  
  Or to find sessions by device type:
  
  ```javascript
  const mobileSessions = sessions.filter(s => 
    s.userAgent.includes('iPhone') || s.userAgent.includes('Android')
  );
  ```
</Tip>
