---
title: "Delete Session"
api: "DELETE /sessions/:id"
description: "Delete a user session"
---

This endpoint permanently deletes a session. This is typically used when a user logs out or when cleaning up expired sessions.

## Authentication

<Info>Authentication requirements depend on your application's security policies.</Info>

## Path Parameters

<ParamField path="id" type="string" required>
  The unique session identifier (UUID) to delete
</ParamField>

## Request Body

<ParamField body="id" type="string" required>
  The session ID (must match the path parameter for confirmation)
</ParamField>

<ParamField body="isActive" type="boolean">
  Optional field - if provided, can be used for additional validation
</ParamField>

## Response

Returns a success message when the session is deleted successfully.

<ResponseField name="message" type="string">
  Confirmation message
</ResponseField>

<ResponseField name="session_id" type="string">
  The ID of the deleted session
</ResponseField>

## Example Request

<CodeGroup>
```bash cURL
curl -X DELETE https://api.rentapelis.com/sessions/987e6543-e21b-98d7-b654-426614174999 \
  -H "Content-Type: application/json" \
  -d '{
    "id": "987e6543-e21b-98d7-b654-426614174999"
  }'
```

```javascript JavaScript
const sessionId = '987e6543-e21b-98d7-b654-426614174999';
const response = await fetch(`https://api.rentapelis.com/sessions/${sessionId}`, {
  method: 'DELETE',
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    id: sessionId
  })
});
const result = await response.json();
console.log(result);
```

```python Python
import requests

session_id = '987e6543-e21b-98d7-b654-426614174999'
headers = {
    'Content-Type': 'application/json'
}

data = {
    'id': session_id
}

response = requests.delete(
    f'https://api.rentapelis.com/sessions/{session_id}',
    headers=headers,
    json=data
)
result = response.json()
print(result)
```
</CodeGroup>

## Use Cases

### User Logout

Delete the current session when a user logs out:

```javascript
// On logout
const currentSessionId = localStorage.getItem('sessionId');
await fetch(`https://api.rentapelis.com/sessions/${currentSessionId}`, {
  method: 'DELETE',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ id: currentSessionId })
});
localStorage.removeItem('sessionId');
localStorage.removeItem('token');
```

### Remote Device Logout

Allow users to remotely log out from specific devices:

```javascript
// Get all user sessions
const sessions = await fetch(`https://api.rentapelis.com/sessions/${userId}`);
const sessionList = await sessions.json();

// Delete a specific session
const sessionToDelete = sessionList.find(s => s.userAgent.includes('iPhone'));
await fetch(`https://api.rentapelis.com/sessions/${sessionToDelete.id}`, {
  method: 'DELETE',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ id: sessionToDelete.id })
});
```

### Logout All Devices

Delete all sessions for a user (useful for security purposes):

```javascript
const sessions = await fetch(`https://api.rentapelis.com/sessions/${userId}`);
const sessionList = await sessions.json();

// Delete all sessions
await Promise.all(
  sessionList.map(session =>
    fetch(`https://api.rentapelis.com/sessions/${session.id}`, {
      method: 'DELETE',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ id: session.id })
    })
  )
);
```

### Cleanup Expired Sessions

Periodically delete expired sessions:

```javascript
const sessions = await fetch(`https://api.rentapelis.com/sessions/${userId}`);
const sessionList = await sessions.json();

const expiredSessions = sessionList.filter(s => 
  new Date(s.expiresAt) < new Date()
);

await Promise.all(
  expiredSessions.map(session =>
    fetch(`https://api.rentapelis.com/sessions/${session.id}`, {
      method: 'DELETE',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ id: session.id })
    })
  )
);
```

## Best Practices

<Tip>
  **Deactivate vs Delete**: Consider whether you need to deactivate or delete a session:
  
  - **Deactivate** (using [Update Session](/api/sessions/update-session)): Keeps the session record for audit purposes but marks it as inactive
  - **Delete**: Permanently removes the session record
  
  For security compliance and audit trails, deactivation is often preferred.
</Tip>

<Warning>
  Deleting a session will immediately invalidate any tokens associated with it. Users will need to log in again to create a new session.
</Warning>

## Security Notes

- Always validate that the user has permission to delete the session
- Consider implementing rate limiting to prevent abuse
- Log session deletions for security auditing
- Implement CSRF protection for session deletion endpoints
