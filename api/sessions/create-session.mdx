---
title: "Create Session"
api: "POST /sessions"
description: "Create a new user session"
---

This endpoint creates a new session for a user. Sessions are used to track user authentication tokens and device information.

## Authentication

<Info>This endpoint is typically called internally during the login process.</Info>

## Request Body

<ParamField body="userId" type="string" required>
  The unique user identifier (UUID)
</ParamField>

<ParamField body="refreshToken" type="string" required>
  The refresh token for the session
</ParamField>

<ParamField body="userAgent" type="string">
  The user agent string of the client (browser/device information)
</ParamField>

<ParamField body="ipAddress" type="string">
  The IP address of the client
</ParamField>

<ParamField body="location" type="string">
  Geographic location of the client (e.g., "Bogotá, Colombia")
</ParamField>

<ParamField body="expiresAt" type="string">
  Session expiration date in ISO 8601 format
</ParamField>

## Response

Returns the created session object with all its properties.

<ResponseField name="id" type="string">
  Unique session identifier (UUID)
</ResponseField>

<ResponseField name="userId" type="string">
  The user ID associated with this session
</ResponseField>

<ResponseField name="refreshToken" type="string">
  The refresh token for this session
</ResponseField>

<ResponseField name="userAgent" type="string">
  The user agent string
</ResponseField>

<ResponseField name="ipAddress" type="string">
  The client's IP address
</ResponseField>

<ResponseField name="location" type="string">
  Geographic location
</ResponseField>

<ResponseField name="isActive" type="boolean">
  Whether the session is currently active (default: true)
</ResponseField>

<ResponseField name="expiresAt" type="string">
  Session expiration timestamp
</ResponseField>

<ResponseField name="createdAt" type="string">
  Session creation timestamp
</ResponseField>

<ResponseField name="lastUsedAt" type="string">
  Last time the session was used
</ResponseField>

## Error Responses

<ResponseField name="statusCode" type="number">
  HTTP status code
</ResponseField>

<ResponseField name="message" type="string | array">
  Error message description or array of validation errors
</ResponseField>

### 400 Bad Request

Validation errors will return an array of messages:

```json
{
  "statusCode": 400,
  "message": [
    "userId must be a UUID",
    "userId should not be empty",
    "refreshToken should not be empty"
  ],
  "error": "Bad Request"
}
```

### 404 Not Found

Returned when the user ID does not exist:

```json
{
  "statusCode": 404,
  "message": "User not found"
}
```

### 500 Internal Server Error

```json
{
  "statusCode": 500,
  "message": "Internal server error"
}
```

## Session Management

<Info>
  Sessions are automatically managed during the authentication flow:
  - Created on login
  - Updated on token refresh
  - Deactivated on logout
  - Deleted when user account is removed (cascade delete)
</Info>
